## Meeting Notes

### SPIR-V Matrix Legalization Strategies

We discussed [llvm/wg-hlsl#446], which proposes a new way to handle
legalization of matrices for SPIR-V. The existing approach tries to expand the
vector-representation of a matrix into powers of two and then split into
4-length vectors, whereas the new approach would use a largest common divisor
between the source and destination. It was generally agreed that the new
approach sounds like an improvement.

### This Week in HLSL

32 PRs were merged this week across several areas:

- **HLSL Frontend**
  - `ConstantBuffer<T>` copying, conversion, and method support.
  - Implemented `Texture2DArray` support.
- **DirectX Backend**
  - Added DXContainer ObjectYAML PRIV part support.
  - Bug fixes regarding code motion of resource pointers.
  - Widened i1 indexable temps to i32 in DXILDataScalarization.
- **HLSL/DXIL Builtins**
  - Implemented `InterlockedOr` and `InterlockedOr64` builtins.
- **SPIR-V Backend**
  - Multiple improvements to lowering and legalization.
- **Offload Test Suite**
  - New tests in the Matrix and ConstantBuffer<T> areas.
  - Further code cleanup to consolidate backend logic.
  - Raytracing improvements for Metal.
  - VK- and DX-only test tags for CI.
  - Fixed issues with stale tests being run.

For a full list of merged PRs, see the appendix.

There are 11 newly opened PRs and 54 currently ongoing PRs.
